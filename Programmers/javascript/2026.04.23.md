## 1단계 복습 — 어제 풀다가 막혔던 `slice`

### 어제의 좌절 포인트

> 배열에서 "인덱스 2개를 받아서 그 사이를 자르는 법"을 까먹어서 답을 봤다.
> `slice`의 존재 자체가 바로 떠오르지 않았음.

### 근데 진짜 헷갈렸던 건 — `slice(a, b)`에서 `b`가 인덱스인가 개수인가?

**정답: 둘 다 인덱스. 끝 인덱스는 미포함.**

```javascript
[10, 20, 30, 40, 50].slice(1, 3);
// 시작 인덱스 1 (포함) → 끝 인덱스 3 (미포함)
// 결과: [20, 30]
```

헷갈리는 이유: 결과 길이가 `끝 - 시작`(= 3 - 1 = 2개)이라서 "두 번째가 개수인가?" 착각이 생김.
실제로는 **둘 다 인덱스**이고, **끝 인덱스만 미포함**이라 그렇게 보이는 것뿐.

### 비슷하게 생겼는데 동작이 다른 애들 — 진짜 주의

| 메서드                       | 두 번째 인자           | 예시                                  |
| ---------------------------- | ---------------------- | ------------------------------------- |
| `slice(start, end)`          | **끝 인덱스 (미포함)** | `[1,2,3,4].slice(1, 3)` → `[2, 3]`    |
| `substring(start, end)`      | **끝 인덱스 (미포함)** | `"abcd".substring(1, 3)` → `"bc"`     |
| `substr(start, length)`      | **개수** (deprecated)  | `"abcd".substr(1, 3)` → `"bcd"`       |
| `splice(start, deleteCount)` | **개수**               | `[1,2,3,4].splice(1, 3)` → `[1]` 남음 |

> `slice`는 인덱스, `splice`는 개수. 한 글자 차이인데 동작이 다름.
> 외울 때: **slice는 "자른다"(원본 그대로) / splice는 "끼어든다"(원본 변경, 개수 지정)** — 이거로 구분.

## 어제 푼 문제 재정리 — 정수 내림차순 배치

### 문제: 정수 내림차순으로 배치하기 ([링크](https://school.programmers.co.kr/learn/courses/30/lessons/12933))

### 1차 풀이 — `Number()` 사용

```javascript
function solution(n) {
  var answer = String(n)
    .split("")
    .sort((a, b) => b - a);
  return Number(answer.join(""));
}
```

### 2차 풀이 — 단항 `+` 연산자로 더 짧게

```javascript
function solution(n) {
  n = String(n)
    .split("")
    .sort((a, b) => b - a)
    .join("");
  return +n;
}
```

> `+"123"` = `Number("123")` = `123`.
> 문자열 앞에 `+` 하나면 숫자 변환됨. 짧게 쓸 수 있어서 코테에서 유용.

## 오늘의 1단계 문제 풀이

### 문제: 푸드 파이트 대회 ([링크](https://school.programmers.co.kr/learn/courses/30/lessons/134240))

> 선수들이 먹는 음식 개수가 주어지면, 가운데 `0`(물)을 기준으로 **좌우 대칭**이 되도록 음식 번호를 배치해서 반환하는 문제.
> 각 음식 `i`는 `food[i] / 2`번씩 양쪽에 놓을 수 있음 (짝이 안 맞으면 버림).

### 풀이 전략 — 주석 먼저 쓰고 풀기

> 주석으로 흐름을 먼저 적고 코드로 옮기니까 훨씬 수월했음.
> 앞으로도 이 방식으로 풀자.

### 풀이 코드

```javascript
function solution(food) {
  //1. food 반복문 (i=1부터, 물은 제외)
  //2. food[i]가 2보다 작으면 패스 (짝으로 못 놓음)
  //3. food[i] / 2 번 만큼 숫자 i를 문자열로 만들어서
  //   first에는 뒤에 붙이고 (→ 오름차순 배치)
  //   second에는 앞에 붙인다 (→ 내림차순 배치, 대칭)
  //4. first + '0' + second 반환

  let first = "";
  let second = "";

  for (const i in food) {
    if (food[i] < 2) continue;

    first = first + String(i).repeat(Math.floor(food[i] / 2));
    second = String(i).repeat(Math.floor(food[i] / 2)) + second;
  }

  return first + "0" + second;
}
```

### 풀이 중 의문 — "asc / desc로 쌓이는 변수"라는 주석이 맞나?

**살짝 부정확했음.** 의도는 맞는데 표현이 어긋남.

코드를 다시 보면 루프는 `i = "1", "2", "3", ...` 으로 **오름차순 한 번만 돈다**.
변수가 asc/desc로 쌓이는 게 아니라, **같은 오름차순 순서를 어느 쪽에 붙이냐의 차이**.

| 변수     | 붙이는 방식                 | 결과                                                   |
| -------- | --------------------------- | ------------------------------------------------------ |
| `first`  | 뒤에 붙임 (`first + 새값`)  | 왼쪽부터 오름차순 → `"1223"`                           |
| `second` | 앞에 붙임 (`새값 + second`) | 오른쪽부터 오름차순 → **결과적으로 내림차순** `"3221"` |

> 핵심: **같은 순서로 순회**하는데 한쪽은 뒤에, 한쪽은 앞에 붙여서 **좌우 대칭**을 만드는 거.
> "asc/desc로 쌓인다"가 아니라 **"대칭으로 쌓인다"** 가 맞는 표현.

### 놓칠 뻔한 엣지 케이스

> `food[0]`은 물이라 항상 건너뛰어야 하는데, 문제 조건상 `food[0] = 1`로 고정이라 `< 2` 조건에 자연스럽게 걸려서 패스됨.
> 운 좋게 통과한 거고, 원래 의도대로라면 `for (let i = 1; i < food.length; i++)`로 시작하는 게 명확함.

## 오늘의 핵심 정리

### `slice` vs `splice` — 헷갈림 주의

```javascript
// slice(시작 인덱스, 끝 인덱스) — 끝은 미포함, 원본 유지
[1, 2, 3, 4, 5].slice(1, 3); // [2, 3]

// splice(시작 인덱스, 삭제 개수) — 원본 변경
const arr = [1, 2, 3, 4, 5];
arr.splice(1, 3); // arr → [1, 5]
```

### 문자열 → 숫자 변환 단축

```javascript
Number("123") + // 123
  "123"; // 123 — 단항 +, 더 짧음
```

### 문제 풀이 습관 — 주석 먼저

> 코드 치기 전에 주석으로 흐름 1~4단계 적기.
> 오늘 푸드 파이트는 이렇게 푸니까 훨씬 깔끔하게 완성됨.
> **앞으로 1단계 이상 문제는 무조건 주석부터.**

### 앞으로 의식할 것

- `replaceAll`, `split('')`만 쓰지 말고 `map`, `includes`도 써보기
- `slice` / `splice` 구분 확실히 (slice = 인덱스, splice = 개수)
- 문제 풀기 전 주석으로 흐름 정리
- 변수 의도를 주석으로 적을 때 **정확하게** — "asc/desc로 쌓인다"처럼 결과와 과정을 혼동하지 말기
