### 1. 대소문자 구분 — `toUpperCase` / `toLowerCase`

[문제: 대문자와 소문자](https://school.programmers.co.kr/learn/courses/30/lessons/120893)

> 자바스크립트에 `isLowerCase` 이딴 거 **없다**.
> 대신 `toUpperCase()` / `toLowerCase()`로 변환한 값과 원본을 비교해서 판별하면 됨.

```javascript
function solution(my_string) {
  var answer = my_string.split("").map((e) => {
    if (e === e.toLowerCase()) {
      return e.toUpperCase();
    } else {
      return e.toLowerCase();
    }
  });
  return answer.join("");
}
```

**판별 로직이 핵심:**

```javascript
e === e.toLowerCase(); // 소문자와 같다? → 원본이 소문자
e === e.toUpperCase(); // 대문자와 같다? → 원본이 대문자
```

> 자바스크립트는 파이썬의 `.islower()` 같은 게 없어서 이렇게 **변환 후 비교** 방식으로 쓴다.
> 외울 것: `toLowerCase()` / `toUpperCase()` — 둘 다 괄호 잊지 말기.

### 2. `Math.max` + `indexOf` — 최댓값과 그 위치

[문제: 최댓값 만들기 (1)](https://school.programmers.co.kr/learn/courses/30/lessons/120899)

### 1차 풀이 — 반복문으로 직접

```javascript
function solution(array) {
  var answer = [];
  var max = 0;

  for (const item of array) {
    if (max < item) {
      max = item;
    }
  }

  answer[0] = max;
  answer[1] = array.findIndex((e) => e === max);

  return answer;
}
```

정답은 맞는데, `Math.max` 쓰면 한 줄로 끝남.

### 2차 풀이 — `Math.max` + `indexOf`

```javascript
function solution(array) {
  var answer = [];

  answer[0] = Math.max(...array);
  answer[1] = array.indexOf(answer[0]);

  return answer;
}
```

### 의문 1 — `Math.max(...array)` 왜 `...` 써야 함?

**`Math.max`는 배열이 아니라 "숫자 여러 개"를 인자로 받게 설계됐기 때문.**

```javascript
Math.max(1, 2, 3); // 3  ✅ 숫자 여러 개 OK
Math.max([1, 2, 3]); // NaN ❌ 배열 통째로 넣으면 안 됨
Math.max(...[1, 2, 3]); // 3  ✅ 펼쳐서 넣어주면 OK
```

> `...` = **전개 연산자 (spread operator)**.
> 배열을 "풀어헤쳐서" 개별 인자로 넣어주는 역할.
> `[1, 2, 3]` → `1, 2, 3` 이렇게 펴줌.
> `Math.max`, `Math.min`처럼 "숫자 여러 개"를 받는 함수에 배열을 넘길 때 필수.

### 의문 2 — `indexOf`가 뭐임?

> **값을 주면 그 값이 있는 인덱스를 찾아서 반환.** 없으면 `-1`.

```javascript
[10, 20, 30]
  .indexOf(20) // 1
  [(10, 20, 30)].indexOf(99); // -1 (없으면 -1)
```

### 주의 — 중복 값이 있으면 **첫 번째 인덱스만** 반환

```javascript
[10, 20, 20, 30].indexOf(20); // 1 (두 번째 20은 무시)
```

> 오늘 헷갈렸던 부분: "배열에 동일 값 있으면 못 찾네" → **못 찾는 게 아니라 첫 번째만 찾음**.
> 이 문제에서는 최댓값이 어차피 첫 번째 위치만 알면 되니까 상관없었음.

### 비슷한 메서드 비교

| 메서드            | 용도                               | 예시                                  |
| ----------------- | ---------------------------------- | ------------------------------------- |
| `indexOf(값)`     | **값**으로 인덱스 찾기 (첫 번째)   | `[1,2,3].indexOf(2)` → `1`            |
| `findIndex(함수)` | **조건**으로 인덱스 찾기 (첫 번째) | `[1,2,3].findIndex(e => e > 1)` → `1` |
| `lastIndexOf(값)` | 값으로 인덱스 찾기 (**마지막**)    | `[1,2,1].lastIndexOf(1)` → `2`        |

> `indexOf`는 단순 값 비교, `findIndex`는 조건이 필요할 때. 오늘처럼 단순히 같은 값 찾는 거면 `indexOf`가 더 깔끔.

### 3. `includes` — 어제 다짐한 거 바로 써먹음

[문제: 가위 바위 보](https://school.programmers.co.kr/learn/courses/30/lessons/120903)

> 어제 "`includes` 의식적으로 쓰자"고 써놓고 처음엔 `map` 이중으로 풀어버림.

### 1차 풀이 — 이중 `map` (비효율)

```javascript
function solution(s1, s2) {
  var answer = 0;

  s1.map((e) => {
    s2.map((e2) => {
      if (e === e2) {
        answer += 1;
      }
    });
  });

  return answer;
}
```

### 2차 풀이 — `filter` + `includes`

```javascript
function solution(s1, s2) {
  var answer = s1.filter((e) => s2.includes(e));
  return answer.length;
}
```

> **`filter` + `includes` 조합은 진짜 자주 쓰임.** 실무에서도 "A 배열 중 B 배열에 포함된 것만" 같은 패턴 많음.
> 어제 다짐한 거 오늘 바로 실전 — 이런 식으로 근육 만들어야 함.

**참고:** `map`은 "변환"할 때 쓰는 거고, 여기서처럼 카운트만 하려고 `map` 쓰는 건 용도가 틀린 거. `forEach`가 더 맞고, 더 나아가서는 애초에 `filter`로 갔어야 함.

### 4. `filter` — 조건에 맞는 것만 뽑기

[문제: 배수 구하기](https://school.programmers.co.kr/learn/courses/30/lessons/120905)

### 1차 풀이 — `for` + `push`

```javascript
function solution(n, numlist) {
  var answer = [];

  for (const num of numlist) {
    if (num % n === 0) {
      answer.push(num);
    }
  }

  return answer;
}
```

### 2차 풀이 — `filter`

```javascript
function solution(n, numlist) {
  return numlist.filter((e) => e % n === 0);
}
```

> **한 줄로 끝.** `filter`는 조건에 맞는 것만 골라서 새 배열로 반환.
> "if로 push" 패턴이 보이면 → **`filter` 신호탄**.

### 패턴 암기

| 하고 싶은 것              | 쓸 메서드  |
| ------------------------- | ---------- |
| 각 원소를 **변환**        | `map`      |
| 조건에 맞는 것만 **뽑기** | `filter`   |
| 포함 여부 **확인**        | `includes` |
| 다 합쳐서 **하나로**      | `reduce`   |

## 1단계 — 오늘의 보스

### 문제: 두 개 뽑아서 더하기

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/68644)

> 배열에서 서로 다른 두 개를 뽑아 더한 모든 경우의 수를 **중복 없이 오름차순**으로 반환.
> 1시간 걸림. GPT 도움도 받음. 근데 결국 이해하고 다시 풀었으니 OK.

### 최종 풀이 — `includes`로 중복 거르기

```javascript
function solution(numbers) {
  var answer = [];

  for (var i = 0; i < numbers.length; i++) {
    for (var j = i + 1; j < numbers.length; j++) {
      if (!answer.includes(numbers[i] + numbers[j])) {
        answer.push(numbers[i] + numbers[j]);
      }
    }
  }

  return answer.sort((a, b) => a - b);
}
```

### 핵심 — 왜 `j = i + 1`인가?

> **같은 인덱스 쌍을 두 번 세지 않기 위해서.**

```
numbers = [2, 1, 3, 4, 1]
           0  1  2  3  4

i=0일 때 j=1,2,3,4 → (2+1), (2+3), (2+4), (2+1)
i=1일 때 j=2,3,4   → (1+3), (1+4), (1+1)
i=2일 때 j=3,4     → (3+4), (3+1)
i=3일 때 j=4       → (4+1)
```

> `j`가 `i+1`부터 시작해야 `(0,1)`과 `(1,0)` 같은 중복 쌍을 안 만든다.
> **"조합(combination)"의 기본 패턴** — 외워두면 평생 써먹음.

### GPT가 알려준 `new Set` 풀이

```javascript
function solution(numbers) {
  var answer = [];
  for (var i = 0; i < numbers.length; i++) {
    for (var j = i + 1; j < numbers.length; j++) {
      answer.push(numbers[i] + numbers[j]);
    }
  }
  return [...new Set(answer)].sort((a, b) => a - b);
}
```

### `Set`이 뭐냐 — 중복 없는 자료구조

> **`Set`은 중복을 자동으로 제거하는 컬렉션.** 같은 값 두 번 넣어도 하나만 남음.

```javascript
new Set([1, 2, 2, 3, 3, 3]); // Set { 1, 2, 3 }
```

### 사용법 — 배열의 중복 제거 공식

```javascript
const arr = [1, 2, 2, 3, 3, 3];

// 1단계: 배열을 Set으로 → 중복 제거됨
const set = new Set(arr); // Set { 1, 2, 3 }

// 2단계: 다시 배열로 변환 (전개 연산자 사용)
const unique = [...set]; // [1, 2, 3]

// 한 줄로:
const uniqueArr = [...new Set(arr)]; // [1, 2, 3]
```

### `[...new Set(array)]` — **중복 제거 치트키**

> 이 한 줄은 **외워두자**. 자바스크립트 코테에서 엄청 자주 나옴.
> 순서:
>
> 1. `new Set(array)` → 배열을 Set으로 바꿔서 중복 제거
> 2. `[...]` → 다시 배열로 펼침 (`Math.max(...array)`의 그 `...`와 같은 원리)

### 두 풀이 비교

| 방식                                 | 장점           | 단점                                         |
| ------------------------------------ | -------------- | -------------------------------------------- |
| **매번 `includes` 체크**             | 메모리 효율    | `includes`가 매번 배열을 훑어서 느릴 수 있음 |
| **다 `push` 후 `Set`으로 중복 제거** | 코드 짧고 빠름 | 일단 다 넣었다 빼는 구조                     |

> 이 문제 크기(numbers 최대 100개)에서는 둘 다 통과. 근데 **`Set` 방식이 더 실무적**이고 깔끔함.
> `includes`를 반복문 안에서 쓰면 O(n²) → O(n³)로 늘어나니까 데이터 커지면 위험.

## 오늘의 핵심 정리

### 전개 연산자 `...` — 쓰임새 정리

```javascript
// 1. 배열 → 개별 인자로 "펼치기"
Math.max(...[1, 2, 3])       // 3

// 2. Set → 배열로 "펼치기"
[...new Set([1, 1, 2])]      // [1, 2]

// 3. 배열 복사 / 합치기
const b = [...a]             // a 복사
const c = [...a, ...b]       // a와 b 합치기
```

> **`...`는 "펼친다"** 한 단어로 이해하면 끝. 배열이든 Set이든 "풀어헤쳐서" 개별 원소로 만들어줌.

### 값/인덱스 찾기 메서드

```javascript
arr.includes(값); // true / false
arr.indexOf(값); // 인덱스 / -1 (첫 번째만)
arr.findIndex(함수); // 조건으로 인덱스 찾기
```

### `Set`으로 중복 제거 공식

```javascript
[...new Set(arr)]; // 이거 외워!
```

### 조합(Combination) 루프 패턴

```javascript
for (let i = 0; i < arr.length; i++) {
  for (let j = i + 1; j < arr.length; j++) {
    // (i, j) 쌍 — 중복 없는 조합
  }
}
```

> `j = i + 1`이 **핵심 포인트.** 외우자.
