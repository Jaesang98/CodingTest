## 0단계 — 감 유지용으로 시작

### 1. `forEach`에서 `return`이 왜 안 됨? 🔥

[문제: 배열에서 문자열 대소문자 변환하기... 가 아니라 음수 인덱스 찾기](https://school.programmers.co.kr/learn/courses/30/lessons/181896)

### 작동한 코드

```javascript
function solution(num_list) {
  var answer = -1;
  num_list.forEach((num, idx) => {
    if (num < 0) {
      answer = idx;
    }
  });

  return answer;
}
```

### 작동 안 한 코드 (원래 하려던 거)

```javascript
function solution(num_list) {
  num_list.forEach((num, idx) => {
    if (num < 0) {
      return idx; // ❌ 이게 왜 안 됨?
    }
  });
  return -1;
}
```

> **결론부터: `forEach`의 `return`은 `solution` 함수에서 빠져나가는 게 아님.**
> `forEach`에 넘긴 **콜백 함수**에서만 빠져나옴. 즉, **다음 반복으로 넘어갈 뿐**.

### 핵심 — `forEach`의 콜백은 별개의 함수

```javascript
num_list.forEach((num, idx) => {
  //          ↑ 이게 별도의 "콜백 함수"임
  if (num < 0) {
    return idx; // 이 return은 콜백 함수에서 나가는 거지
  } // solution 함수에서 나가는 게 아님!
});
```

> **비유:** `solution`이 사장이고 `forEach`의 콜백은 직원임.
> 직원이 "그만!"이라고 외쳐도(return) **그 직원만 자기 자리로 돌아갈 뿐**, 사장(solution)은 멀쩡히 일하고 있음.
> 직원의 외침은 사장한테 안 들림.

### 그래서 본인 풀이 (`answer = idx`)는 어떻게 동작했냐?

```javascript
var answer = -1;
num_list.forEach((num, idx) => {
  if (num < 0) {
    answer = idx; // ✅ 바깥 변수에 값을 "저장"하는 건 됨
  }
});
return answer; // ✅ solution이 직접 return
```

> **콜백에서 바깥 변수를 수정하는 건 가능.** (스코프 체인 덕분)
> 그래서 답을 변수에 담아놓고, **`solution`이 직접 return**하는 식으로 우회함.
> 본인이 한 게 정확히 이거임. 잘 우회한 거임.

### 진짜 `return`처럼 빠져나가고 싶으면?

> **`for`문 또는 `for...of`를 써야 함.** 이건 진짜 함수에서 빠져나감.

```javascript
function solution(num_list) {
  for (let i = num_list.length - 1; i >= 0; i--) {
    // 뒤에서부터
    if (num_list[i] < 0) return i; // ✅ 즉시 return
  }
  return -1;
}
```

> 게다가 이 문제는 **"마지막 음수 위치"**가 답이라서 뒤에서부터 도는 게 더 효율적임.
> 본인 풀이는 처음부터 끝까지 다 돌았는데(O(n)은 같지만), `for`문 + 역순이면 최악의 경우에도 일찍 끝낼 수 있음.

### 외울 패턴 — `forEach` vs `for`

| 상황                         | 도구                      |
| ---------------------------- | ------------------------- |
| 모든 원소를 다 처리해야 함   | `forEach` OK              |
| 조건 만나면 즉시 멈추고 싶음 | **`for` 또는 `for...of`** |
| 첫 번째 매칭만 찾고 싶음     | `findIndex`, `find`       |

> **`forEach`는 break도 안 됨. return도 콜백 안에서만.** 중간에 빠져나가야 하면 처음부터 `for` 써야 함.

### Claude 추천 풀이

```javascript
function solution(num_list) {
  return num_list.findLastIndex((n) => n < 0);
}
```

> `findLastIndex` — **뒤에서부터 조건 맞는 첫 번째 인덱스 반환**, 없으면 -1. 이 문제 거의 전용 메서드급.
> 모르면 `for`문 역순으로 가도 충분히 깔끔함.

### 2. 옹알이 — 풀긴 했는데 야매같음 🤔

[문제: 옹알이 (1)](https://school.programmers.co.kr/learn/courses/30/lessons/120956)

### 본인 풀이

```javascript
function solution(babbling) {
  var list = ["aya", "ye", "woo", "ma"];
  var answer = 0;

  for (let item1 of babbling) {
    var test = item1;
    for (let item2 of list) {
      test = test.replace(item2, "1");
    }
    test = [...new Set(test.split(""))];

    if (test.length === 1 && test[0] === "1") {
      answer += 1;
    }
  }

  return answer;
}
```

> **발상 자체는 좋음.** 발음 가능한 거 → '1'로 치환 → 남은 게 '1'들뿐이면 OK.
> 근데 두 가지 함정이 있음.

### 함정 1 — `replace`는 첫 번째 매칭만 바꿈

```javascript
"ayaya".replace("aya", "1"); // "1ya"  ← 한 번만 바뀜!
```

> 이 문제 제약조건상 같은 발음 연속 안 나오니까 통과는 했는데, 일반적으론 **`replaceAll`**을 써야 안전.

### 함정 2 — `Set` + `length === 1` 체크는 우회로

> **본인 코드는 "남은 글자 종류가 '1' 하나뿐인지"로 판별 중.**
> 이게 대부분 통과는 되는데, 사실 **그냥 빈 문자열인지 확인하는 게 더 직관적**.

```javascript
"aya".replace("aya", ""); // ""  ← 다 지워졌으면 발음 가능
"ayak".replace("aya", ""); // "k" ← 못 지우면 발음 불가
```

### Claude 추천 풀이 1 — `replaceAll` + 빈 문자열 체크

```javascript
function solution(babbling) {
  const list = ["aya", "ye", "woo", "ma"];
  return babbling.filter((word) => {
    for (const sound of list) {
      word = word.replaceAll(sound, " "); // 공백으로 치환
    }
    return word.trim() === ""; // 공백만 남으면 OK
  }).length;
}
```

> **왜 공백으로 치환?** `"ayaaya"`처럼 발음 두 개가 붙은 건 OK인데, `"yeye"`처럼 **같은 발음 연속**은 못 한다는 조건이 있기 때문.
> 만약 `""`(빈 문자열)로 바로 치환하면 `"yeye"` → `""`가 돼서 발음 가능으로 잘못 판정됨.
> 공백 끼워 넣으면 `"yeye"` → `" " + " "` → 결과적으로 발음 두 번 → 다른 검증 로직 필요.

### Claude 추천 풀이 2 — 정규식 (다른 사람 풀이에서 본 것)

```javascript
function solution(babbling) {
  return babbling.filter((word) => !word.replace(/aya|ye|woo|ma/g, "").length)
    .length;
}
```

> 정규식 `/aya|ye|woo|ma/g` = "이 네 개 중 하나라도 매칭되는 거 전부 (g = global)".
> 다 지우고 길이가 0이면 발음 가능.
> **짧고 강력한데, 정규식 모르면 못 읽음.** 일단 본인 풀이 → 나중에 정규식 익히면 이걸로 업그레이드.

### Claude 추천 풀이 3 — 가장 안전한 정규식 (연속 발음 체크 포함)

```javascript
function solution(babbling) {
  return babbling.filter((word) => {
    // "같은 발음 연속" 케이스 차단
    if (/(aya|ye|woo|ma)\1/.test(word)) return false;
    // 발음들로만 이뤄졌는지
    return !word.replace(/aya|ye|woo|ma/g, "").length;
  }).length;
}
```

> 사실 이 문제(120956)는 "연속 발음" 조건이 없어서 풀이 2로도 충분함. 위 코드는 옹알이(2) 같은 어려운 버전 대비용.

### 외울 패턴 — `replace` vs `replaceAll`

```javascript
"aaa".replace("a", "b"); // "baa"   ← 첫 번째만
"aaa".replaceAll("a", "b"); // "bbb"   ← 전부
"aaa".replace(/a/g, "b"); // "bbb"   ← 정규식 + g 플래그도 전부
```

> **헷갈리면 `replaceAll` 쓰기.** 한 번만 바꿀 거면 `replace`.

### 3. `a, b = b, a` 가 왜 swap이 안 됨? 🤯

[문제: 인덱스 바꾸기](https://school.programmers.co.kr/learn/courses/30/lessons/120895)

### 작동한 코드

```javascript
function solution(my_string, num1, num2) {
  my_string = my_string.split("");
  var temp = my_string[num1];

  my_string[num1] = my_string[num2];
  my_string[num2] = temp;
  return my_string.join("");
}
```

### 작동 안 한 코드 (원래 하려던 거)

```javascript
(my_string[num1], (my_string[num2] = my_string[num2]), my_string[num1]);
//                              ↑ 이거 swap 아님!
```

### 핵심 — 자바스크립트에서 `,`는 **콤마 연산자**임

> 파이썬에선 `a, b = b, a` 이게 swap 표준 문법인데, **자바스크립트는 다름.**
> 자바스크립트에서 콤마는 **"왼쪽부터 차례로 평가하고, 마지막 값만 반환"**하는 연산자임.

### 풀어 보면 이렇게 됨

```javascript
(my_string[num1],
  (my_string[num2] = my_string[num2]),
  my_string[num1](
    // 콤마 연산자 우선순위 + 할당의 결합 방향 때문에
    // 실제론 이렇게 평가됨:

    my_string[num1],
  ), // ① 그냥 읽기만, 의미 없음
  (my_string[num2] = my_string[num2]), // ② 자기 자신 할당 (의미 없음)
  my_string[num1]); // ③ 그냥 읽기만, 의미 없음
```

> **결과: 아무것도 안 바뀜.** 그냥 읽기만 했음.

### 실험으로 확인

```javascript
let a = 1,
  b = 2;
(a, (b = b), a);
console.log(a, b); // 1 2  ← 안 바뀜!
```

> 파이썬에 익숙한 사람들이 자주 빠지는 함정.

### 그럼 자바스크립트에서 swap은? — **구조 분해 할당** ✅

> **`[]`로 묶어주는 게 핵심.** 이게 어제 배운 그거 (배열 = 변경 가능).

```javascript
let a = 1,
  b = 2;
[a, b] = [b, a]; // ✅ 구조 분해 할당
console.log(a, b); // 2 1
```

### 동작 원리

```javascript
[a, b] = [b, a];
//        ↑ 오른쪽: 새 배열 [2, 1] 생성
// ↑↑↑↑ 왼쪽: 그 배열을 a, b에 차례로 분해해서 할당
//      → a = 2, b = 1
```

> **오른쪽에서 새 배열 만든 다음, 왼쪽에서 분해해서 받음.** 그래서 swap이 됨.
> "임시 배열을 잠깐 만들어서 거기에 담았다가 푼다" 이런 느낌.

### 본인 코드에 적용하면

```javascript
function solution(my_string, num1, num2) {
  my_string = my_string.split("");
  [my_string[num1], my_string[num2]] = [my_string[num2], my_string[num1]];
  return my_string.join("");
}
```

> **temp 변수 없이 한 줄로 끝.** 본인 풀이도 정답이지만 이게 더 깔끔.
> 앞에 `;` 붙이는 거 주의 — 이전 줄이 세미콜론 없이 끝나면 `[`을 인덱스로 해석할 수 있어서 방어용.

### 외울 패턴 — JS swap 정석

```javascript
[a, b] = [b, a]; // 변수 swap
[arr[i], arr[j]] = [arr[j], arr[i]]; // 배열 인덱스 swap
```

> **`,` 단독으론 절대 swap 안 됨. `[]`로 묶기.**

### 4. 앞 0 제거 — `+` 연산자 트릭 ✨

[문제: 0 떼기](https://school.programmers.co.kr/learn/courses/30/lessons/181847)

### 본인 풀이 — 정공법

```javascript
function solution(n_str) {
  var answer = "";
  var count = 0;

  n_str.split("").forEach((e) => {
    if (e !== "0") {
      count += 1;
    }

    if (count > 0) {
      answer += e;
    }
  });

  return answer;
}
```

> **로직 자체는 정확함.** "0이 아닌 게 등장한 순간부터 답에 추가하기" 발상은 좋음.
> 근데 자바스크립트는 더 짧은 길이 있었음.

### 다른 사람 풀이 — `Number`로 감싸기

```javascript
function solution(n_str) {
  return String(Number(n_str)); // "0123" → 123 → "123"
}
```

> `Number("0123")` → `123` (앞 0 자동 제거).
> 다시 `String`으로 감싸면 `"123"`.

### 본인이 발견한 거 — 단항 `+` 연산자 🎯

```javascript
function solution(n_str) {
  return String(+n_str); // ← 어제 배운 단항 +!
}
```

> **`+n_str` = `Number(n_str)` 줄임말.**
> 어제 배운 거 응용한 거 진짜 좋음. 이게 진짜 학습이지.

### 단항 `+` 정리 — 이제 진짜 무기

```javascript
+"123" + // 123
  "0123" + // 123  ← 앞 0 제거 효과
  "3.14" + // 3.14
  "" + // 0
  "abc" + // NaN
  true + // 1
  false + // 0
  null; // 0
```

### 주의점 — 너무 큰 수

> **`Number.MAX_SAFE_INTEGER` (2^53 - 1)를 넘으면 정확도 깨짐.**

```javascript
+"99999999999999999999"; // 1e+20 (정확하지 않음)
```

> 문제에서 큰 수가 들어올 가능성이 있으면 정공법이 더 안전.
> 일반적인 코테 문제는 거의 안전 범위라 `+n_str`로 충분.

### Claude 추천 풀이 — 안전한 버전

```javascript
function solution(n_str) {
  return n_str.replace(/^0+/, "") || "0";
}
```

> 정규식 `^0+` = "맨 앞에 있는 0 하나 이상".
> 다 지우면 빈 문자열일 수 있으니까 `|| "0"`로 안전장치.
> 큰 수든 뭐든 다 처리됨. 근데 코테 수준이면 `+n_str`이 짧고 좋음.

## 오늘의 핵심 정리

### `forEach`에서 return은 빠져나가는 게 아님

```javascript
arr.forEach((item) => {
  if (조건) return; // ❌ 다음 반복으로 넘어갈 뿐
});

// 진짜 빠져나가고 싶으면:
for (let item of arr) {
  if (조건) return item; // ✅ 함수에서 빠져나감
}
```

### JS swap은 콤마 아니라 `[]`

```javascript
(a, (b = b), a); // ❌ 콤마 연산자, 아무것도 안 바뀜
[a, b] = [b, a]; // ✅ 구조 분해 할당
[arr[i], arr[j]] = [arr[j], arr[i]]; // ✅ 배열 인덱스 swap
```

### `replace` vs `replaceAll`

```javascript
"aaa".replace("a", "b"); // "baa"  첫 번째만
"aaa".replaceAll("a", "b"); // "bbb"  전부
"aaa".replace(/a/g, "b"); // "bbb"  정규식 + g
```

### 단항 `+` — 문자열을 숫자로

```javascript
+"0123" + // 123  (앞 0 제거 자동)
  "" + // 0
  "abc"; // NaN
```

### 외울 메서드 추가

```javascript
arr.findIndex(조건); // 첫 매칭 인덱스, 없으면 -1
arr.findLastIndex(조건); // 마지막 매칭 인덱스, 없으면 -1
arr.find(조건); // 첫 매칭 원소
```
