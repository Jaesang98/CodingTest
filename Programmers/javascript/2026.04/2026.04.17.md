## 1단계 문제 풀이 회고

### 문제: 숫자 문자열과 영단어 ([링크](https://school.programmers.co.kr/learn/courses/30/lessons/81301))

**소요 시간:** 총 약 70~80분

### 풀이 흐름

총 4차 피드백을 거쳐서 아래처럼 발전했어.

| 버전    | 핵심 변화                                                               |
| ------- | ----------------------------------------------------------------------- |
| 처음    | 재귀 함수(`jaguie`) 사용, `if(parseInt(result))`로 불완전하게 반복      |
| 1차     | `if` 조건 제거하고 무조건 반복 — 그래도 `replace`는 첫 번째 일치만 교체 |
| 2차     | `while` + `includes`로 모든 영단어 교체 보장                            |
| 3차     | `replaceAll`로 단순화                                                   |
| **4차** | `split(a).join(값)`으로 교체 — 가장 깔끔한 최종 풀이                    |

### 처음 — 재귀 함수 사용

> `if(parseInt(result))`로 조건을 걸었는데, 이미 숫자가 섞여 있으면 조건이 true가 되어버려서 의도한 대로 동작하지 않음.

```javascript
function solution(s) {
  var result = s;
  var Numbers = [
    "zero",
    "one",
    "two",
    "three",
    "four",
    "five",
    "six",
    "seven",
    "eight",
    "nine",
  ];

  result = jaguie(result);

  Numbers.forEach((e) => {
    if (parseInt(result)) {
      result = jaguie(result);
    }
  });

  return parseInt(result);
}

function jaguie(result) {
  var Numbers = [
    "zero",
    "one",
    "two",
    "three",
    "four",
    "five",
    "six",
    "seven",
    "eight",
    "nine",
  ];
  var dNumbers = {
    zero: 0,
    one: 1,
    two: 2,
    three: 3,
    four: 4,
    five: 5,
    six: 6,
    seven: 7,
    eight: 8,
    nine: 9,
  };

  Numbers.forEach((e) => {
    result = result.replace(e, dNumbers[e]);
  });

  return result;
}
```

### 1차 피드백 — `if` 조건 제거

> `if` 없애고 무조건 반복하도록 수정. 하지만 `replace`는 **첫 번째 일치만** 교체하기 때문에 같은 단어가 여러 번 나오면 누락됨.

function solution(s) {

    "eight",
    "nine",

  result = jaguie(result);

### 2차 피드백 — `while` + `includes`로 전체 교체 보장

> 함수 이름도 의미 있게 바꾸고, `while(result.includes(e))`로 해당 단어가 없어질 때까지 반복. 모든 케이스 교체 보장.

```javascript
function solution(s) {
  return Number(convertStringWordtoNumber(s));
}

function convertStringWordtoNumber(result) {
  var Numbers = [
    "zero",
    "one",
    "two",
    "three",
    "four",
    "five",
    "six",
    "seven",
    "eight",
    "nine",
  ];
  var dNumbers = {
    zero: 0,
    one: 1,
    two: 2,
    three: 3,
    four: 4,
    five: 5,
    six: 6,
    seven: 7,
    eight: 8,
    nine: 9,
  };

  Numbers.forEach((e) => {
    while (result.includes(e)) {
      result = result.replace(e, dNumbers[e]);
    }
  });

  return result;
}
```

---

### 3차 피드백 — `replaceAll`로 단순화

> `while` 루프 없이 `replaceAll` 한 줄로 동일한 효과.

```javascript
Numbers.forEach((e) => {
  result = result.replaceAll(e, dNumbers[e]);
});
```

### 4차 피드백 (최종) — `split().join()`으로 딕셔너리만 사용

> `Numbers` 배열 없이 `dNumbers` 하나만으로 `for...in` 순회. `split().join()`으로 전체 치환.

```javascript
function solution(s) {
  return Number(convertStringWordtoNumber(s));
}

function convertStringWordtoNumber(result) {
  var dNumbers = {
    zero: 0,
    one: 1,
    two: 2,
    three: 3,
    four: 4,
    five: 5,
    six: 6,
    seven: 7,
    eight: 8,
    nine: 9,
  };

  for (var a in dNumbers) {
    result = result.split(a).join(dNumbers[a]);
  }

  return result;
}
```

## 오늘의 핵심 개념 정리

### `for (var a in 객체)` — 객체 순회

```javascript
for (var a in dNumbers) {
  console.log(a); // "zero", "one", "two" ... 키(key)만 출력됨
}
```

> `for...in`은 객체의 **키(key)만** 순회한다. 값은 `dNumbers[a]`로 접근.

---

### `split` / `join` 콤보 — 문자열 전체 치환 트릭

```javascript
"onetwone".split("one").join(1);
// split → ["", "tw", ""] (배열)
// join(1) → "1tw1"   (문자열)
```

| 메서드         | 입력   | 출력   |
| -------------- | ------ | ------ |
| `split(기준)`  | 문자열 | 배열   |
| `join(연결자)` | 배열   | 문자열 |

> `replace`는 **첫 번째 일치만** 교체하지만, `split().join()`은 **전체를 한 번에** 교체한다. `replaceAll`과 동일한 효과!
