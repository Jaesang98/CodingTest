### 1. `Math.pow` & `Math.abs` — 직접 구현 말고 함수 쓰자 🆕

[문제: 제곱수 / 절댓값 분기 문제](https://school.programmers.co.kr/learn/courses/30/lessons/181839)

> **`Math.sqrt`로 시작했다가 멘붕.** "어 이거 제곱이 아니라 제곱근이잖아?" 하고 깨달음.
> 전에 한 번 배운 건데 또 헷갈림. **반복해서 박아야 함.**

### 처음 헷갈린 거

```javascript
Math.sqrt(9); // 3   ← 제곱근 (square root)
Math.pow(3, 2); // 9   ← 거듭제곱 (power)
```

> **이름으로 외우기:**
>
> - `sqrt` = **s**quare **r**oo**t** (제곱**근**)
> - `pow` = **pow**er (거듭제**곱**)

### 내가 푼 코드

```javascript
function solution(a, b) {
  if (a % 2 !== 0 && b % 2 !== 0) {
    return a * a + b * b;
  } else if (a % 2 !== 0 || b % 2 !== 0) {
    return 2 * (a + b);
  } else {
    return a > b ? a - b : b - a;
  }
}
```

### 함수로 다시 쓴다면

```javascript
function solution(a, b) {
  if (a % 2 !== 0 && b % 2 !== 0) {
    return Math.pow(a, 2) + Math.pow(b, 2); // 또는 a**2 + b**2
  } else if (a % 2 !== 0 || b % 2 !== 0) {
    return 2 * (a + b);
  } else {
    return Math.abs(a - b); // ← 삼항연산자보다 훨씬 깔끔
  }
}
```

### 핵심 — `Math.abs`는 진짜 무기임

> **`a > b ? a - b : b - a`** 이거 **`Math.abs(a - b)`** 한 줄로 끝.
> 두 수의 차이 구할 때 무조건 `abs` 쓰는 게 정석. 외우자.

### `Math.pow` vs `**` (거듭제곱 연산자)

```javascript
Math.pow(2, 10); // 1024
2 ** 10; // 1024  ← 더 짧음, ES2016+
Math.pow(2, 0.5); // 1.414... (제곱근도 가능)
2 ** 0.5; // 1.414...
```

> **외울 패턴:**
>
> - 절댓값 → `Math.abs(x)`
> - 거듭제곱 → `Math.pow(a, b)` 또는 `a ** b`
> - 제곱근 → `Math.sqrt(x)` 또는 `x ** 0.5`
> - **이름 헷갈리지 말 것: sqrt = 루트, pow = 거듭제곱**

### 2. `slice(-n)` — 음수 인덱스가 먹는다고?? 🆕

[문제: 뒤에서 자르기](https://school.programmers.co.kr/learn/courses/30/lessons/181910)

> **충격.** `arr[-1]` 같은 거 쓰면 항상 `undefined` 떠서 **JS는 음수 인덱스 안 먹는 줄 알았음.**
> 근데 `slice(-n)`은 **됨.** 뭐야 이거.

### 내가 처음 쓴 코드

```javascript
return my_string.slice(my_string.length - n, my_string.length);
```

### 더 짧게 가능한 코드

```javascript
return my_string.slice(-n); // ← 끝에서 n개
```

### 핵심 — `slice`는 음수 인덱스 지원함

```javascript
const s = "abcdefg";

s.slice(-1); // "g"        ← 뒤에서 1개
s.slice(-3); // "efg"      ← 뒤에서 3개
s.slice(-3, -1); // "ef"       ← 뒤에서 3번째 ~ 뒤에서 1번째 직전
s.slice(2, -1) // "cdef"     ← 2번 ~ 끝에서 1번째 직전

  [
    // 배열도 똑같음
    (1, 2, 3, 4, 5)
  ].slice(-2); // [4, 5]
```

### 근데 왜 `s[-1]`은 `undefined`?

> **대괄호 인덱싱은 음수 안 먹음.** `slice`만 됨.
> 이유: `s[-1]`은 자바스크립트 입장에서 "키 이름이 `-1`인 프로퍼티 찾기"라서 `undefined`.

```javascript
"abc"[-1]; // undefined
"abc".slice(-1); // "c"      ← 됨
"abc".at(-1); // "c"      ← ES2022, 대괄호 대용으로 음수 가능
```

> **외울 패턴:**
>
> - 한 글자/한 요소 뒤에서 → `.at(-1)`
> - 여러 개 잘라내기 → `.slice(-n)`
> - **대괄호 `[-1]`은 절대 안 됨, 다른 메서드 써야 함**

> **회고:** "아 씨발" 적어놨는데, 이게 진짜 **유레카 신호**임 ㅋㅋ. 모르던 게 풀린 순간 = 가장 잘 박힘.

### 3. 문자열 정렬 — `localeCompare` vs `(a-b)` 왜 안되지?? 🤔

[문제: 대소문자 통일 + 사전순 정렬](https://school.programmers.co.kr/learn/courses/30/lessons/120911)

### 내가 푼 코드

```javascript
function solution(my_string) {
  var answer = my_string
    .split("")
    .map((e) => (e === e.toUpperCase() ? e.toLowerCase() : e));
  answer.sort((a, b) => a.localeCompare(b));
  return answer.join("");
}
```

### `toUpperCase() ? toLowerCase()` 트릭

> **어제 회고에서 본 거 기억나서 씀.** ← 이게 **복습 효과의 증거.**
> 어제 "최근 회고 5분 훑기" 다짐한 게 바로 효과 봄.

```javascript
e === e.toUpperCase() ? e.toLowerCase() : e;
// "이미 대문자면 소문자로, 아니면 그대로"
```

### `localeCompare` — 기억 안 나서 결국 검색함 😤

> **알긴 아는데 안 쓰니까 까먹음.** 자주 쓰는 패턴 노트에 박아놔야 함.

```javascript
// 문자열 사전순 정렬
arr.sort((a, b) => a.localeCompare(b)); // 오름차순
arr.sort((a, b) => b.localeCompare(a)); // 내림차순
```

### 🔥 핵심 의문 — `(a - b)`는 왜 안 되지?

> 본인 질문: **"`return (a - b) > (b - a)` 이런 것도 됐던 거 같은데 왜 안 되지?"**

### 답 — 문자열에서 `a - b` 하면 `NaN`

```javascript
"apple" - "banana"; // NaN  (문자열 빼기는 NaN)
3 -
  (1)[ // 2    (숫자는 됨)
    // 그래서 sort 비교 함수에서:
    (1, 2, 3)
  ]
    .sort((a, b) => a - b) // ✅ 숫자 → 정렬됨
    [("c", "a", "b")].sort((a, b) => a - b); // ❌ 모두 NaN → 정렬 안 됨 (브라우저별로 결과 다름)
```

### 그럼 어제 어떻게 됐었지?

> **착각이거나, 숫자 문자열 ("1", "2", "3") 정렬할 때였을 가능성 높음.**
> 숫자 문자열은 `-` 연산 시 자동으로 숫자로 변환되거든.

```javascript
"3" - "1"; // 2   ← 자동 숫자 변환됨
"c" - "a"; // NaN ← 알파벳은 안 됨
```

### 정렬 비교 함수 — 정리

| 정렬 대상       | 비교 함수                          |
| --------------- | ---------------------------------- |
| 숫자 배열       | `(a, b) => a - b` (오름차순)       |
| 숫자 배열       | `(a, b) => b - a` (내림차순)       |
| 문자열 배열     | `(a, b) => a.localeCompare(b)`     |
| 문자열 (간단히) | `(a, b) => a < b ? -1 : 1` 도 가능 |

> **외울 패턴:**
>
> - **숫자 = 빼기**, **문자열 = `localeCompare`**
> - 헷갈리면 그냥 `localeCompare` 쓰기 (숫자 문자열도 됨, 약간 느릴 뿐)

### `> (b - a)` 부분도 잘못된 이유

```javascript
sort((a, b) => a - b > b - a);
// → true/false 반환 → 1/0으로 변환 → 정렬이 제대로 안 됨
```

> **sort 비교 함수는 음수/0/양수를 반환해야 함.** boolean(true=1, false=0) 반환하면 **불완전한 정렬**.
> 그래서 어제 "됐던 것 같다"는 건 **운이 좋았던 케이스**일 가능성 높음.

### 4. x만큼 간격이 있는 n개의 숫자 — 비효율로 풀었음 😅

[문제: x만큼 간격이 있는 n개의 숫자](https://school.programmers.co.kr/learn/courses/30/lessons/12954)

### 내 코드 (비효율)

```javascript
function solution(x, n) {
  var answer = [];
  var temp = 0;

  for (var i = 1; i <= n; i++) {
    temp += x; // ← 매번 더하기
    answer.push(temp);
  }

  return answer;
}
```

### 더 깔끔한 코드

```javascript
function solution(x, n) {
  var answer = [];
  for (var i = 1; i <= n; i++) {
    answer.push(x * i); // ← 그냥 곱하기
  }
  return answer;
}
```

### 한 줄로 (참고용)

```javascript
const solution = (x, n) => Array.from({ length: n }, (_, i) => x * (i + 1));
```

### 회고 — 왜 비효율로 풀었나?

> **"누적"이라는 말에 꽂혀서 변수 만들고 더하는 패턴**으로 감.
> 근데 등차수열은 **i번째 = x × i** 라는 공식 한 줄이면 끝.
> **"문제 보자마자 누적 패턴 박지 말고, 공식이 있는지 1초만 생각하기."**

### 6. 두 정수 사이의 합 — 삼항으로 한방 정리 🎯

[문제: 두 정수 사이의 합](https://school.programmers.co.kr/learn/courses/30/lessons/12912)

### 내 코드

```javascript
for (var i = a < b ? a : b; i <= (a < b ? b : a); i++) {
  // ...
}
```

> **본인 멘트: "나처럼 쓴 애가 없네 낫배드?? 클로드야 평가해라."** ㅋㅋㅋ

### 클로드의 솔직한 평가

> **결론: 작동은 하는데 추천은 안 함.** 이유는 아래.

**👍 좋은 점**

- 변수 추가 안 하고 한 줄로 처리한 발상은 좋음
- 삼항연산자 두 번 써서 min/max 구한 건 깔끔한 사고

**👎 아쉬운 점**

- **`a < b ? ... : ...`을 두 번 쓰는 게 중복** → 비교 연산이 두 번 들어감
- 가독성 떨어짐. 처음 보는 사람은 "어 뭐지?" 함
- `Math.min` / `Math.max` 쓰면 의도가 즉시 보임

### 더 추천하는 방식

```javascript
function solution(a, b) {
  let sum = 0;
  for (let i = Math.min(a, b); i <= Math.max(a, b); i++) {
    sum += i;
  }
  return sum;
}
```

### 더 더 깔끔하게 (수학 공식)

```javascript
function solution(a, b) {
  const min = Math.min(a, b);
  const max = Math.max(a, b);
  return ((min + max) * (max - min + 1)) / 2; // 등차수열 합 공식
}
```

> **점수: 65/100.** 발상은 흙 속의 진주, 표현은 좀 더 다듬어야.
> **`Math.min` / `Math.max` 두 함수 머리에 박아두기.** 두 수 비교는 거의 항상 이거 씀.

### 7. 문자열 내 p와 y의 개수 — 패턴 짚기

[문제: 문자열 내 p와 y의 개수](https://school.programmers.co.kr/learn/courses/30/lessons/12916)

> 양치기 중에 푼 거. 패턴: `toLowerCase()` → `split('')` → `filter` 카운트.

```javascript
function solution(s) {
  s = s.toLowerCase();
  const p = s.split("").filter((c) => c === "p").length;
  const y = s.split("").filter((c) => c === "y").length;
  return p === y;
}
```

### 더 짧게

```javascript
const solution = (s) =>
  [...s.toLowerCase()].filter((c) => c === "p").length ===
  [...s.toLowerCase()].filter((c) => c === "y").length;
```

> **외울 패턴:** 대소문자 무시 비교 → **무조건 처음에 `toLowerCase()` 박고 시작.**

## 🆕 substring — 한번 익혀보자

> 본인이 마지막에 "substring도 익혀보자" 적었으니까 정리.

### `slice` vs `substring` vs `substr`

```javascript
const s = "abcdefg";

// slice (가장 권장) — 음수 가능
s.slice(1, 4); // "bcd"
s.slice(-3); // "efg"   ← 음수 OK
s.slice(2); // "cdefg"

// substring — 음수 안 됨, 음수 넣으면 0으로 처리
s.substring(1, 4); // "bcd"
s.substring(-3); // "abcdefg"  ← 음수 무시됨 (0으로 처리)
s.substring(4, 1); // "bcd"      ← 작은 수가 앞으로 자동 swap (특이함)

// substr (deprecated, 쓰지 마) — (시작, 길이)
s.substr(1, 3); // "bcd"   ← 끝 인덱스가 아니라 "길이"
```

### 차이 정리

| 메서드                  | 인자           | 음수 지원   | 비고                       |
| ----------------------- | -------------- | ----------- | -------------------------- |
| `slice(start, end)`     | 시작, 끝       | ✅          | **이거 하나만 알면 됨**    |
| `substring(start, end)` | 시작, 끝       | ❌ (0 처리) | 인자 swap 자동             |
| `substr(start, length)` | 시작, **길이** | ✅          | **deprecated, 쓰지 말 것** |

### 그래서 결론

> **그냥 `slice` 하나만 외우자.**
>
> - 음수 됨 (뒤에서부터)
> - 다른 두 개랑 헷갈릴 일 없음
> - 배열에도 똑같이 씀 (메서드 일관성)
>
> `substring`은 "있다는 것만" 알아두기. 실전에서 굳이 쓸 이유 없음.

## 🤔 오늘의 회고 — "양치기 데이"는 효과가 있었나?

### 컨디션 안 좋을 때의 전략 — 나쁘지 않았음

> **결론: 오늘 같은 날엔 양치기 정답.**
>
> 머리 안 돌아가는데 어려운 거 붙잡으면 → 좌절 → 멘탈 박살 → 내일까지 영향.
> 차라리 **쉬운 거 양치기 = 손에 익은 패턴 반복 = 손이 기억함**.

### 오늘 한 양치기의 진짜 가치

| 활동                  | 효과                                         |
| --------------------- | -------------------------------------------- |
| 1단계 1페이지 양치기  | **손 패턴 강화** (느려진 반응속도 회복)      |
| 어제 회고 훑기        | **어제 트릭 다시 만남** (`toUpperCase` 트릭) |
| `slice(-n)` 발견      | **새 무기 획득** ✨                          |
| 자기 코드에 평가 요청 | **메타인지 + 객관화**                        |

> **양치기가 무의미한 게 아님.** "패턴 자동화 훈련"임.
> 단, **양치기만 하면 늘지 않음.** 새 패턴 발견(예: `slice(-n)`)이 같이 있어야 OK.

### 🆕 새로 배운 거

- `Math.abs(a - b)` — 두 수의 차이는 무조건 이거 (삼항연산자 X)
- `Math.pow(a, b)` / `a ** b` — 거듭제곱 (sqrt랑 헷갈리지 말 것)
- `slice(-n)` — **음수 인덱스 됨!** (배열도 동일)
- `.at(-1)` — 한 요소 뒤에서 가져올 때
- `localeCompare` — 문자열 정렬은 이거. 숫자 정렬은 `(a-b)`
- `substring` — 존재만 알아두기, 실전은 `slice`로 통일

### 🔁 다시 박은 패턴

- `e === e.toUpperCase() ? e.toLowerCase() : e` — 대문자 → 소문자 변환
- `toLowerCase()` 후 비교 — 대소문자 무시 처리
- `Math.min` / `Math.max` 두 수 비교
