### 1. `slice` 끝 인덱스 생략하면 끝까지 — 길이 안 써도 됨 ✨

[문제: 5명씩 자르기 (정렬 후 슬라이스)](https://school.programmers.co.kr/learn/courses/30/lessons/181852?language=javascript)

### 처음 푼 코드

```javascript
function solution(num_list) {
  num_list.sort((a, b) => a - b);
  return num_list.slice(5, num_list.length);
}
```

### 더 짧게 — 끝 인덱스 생략

```javascript
function solution(num_list) {
  num_list.sort((a, b) => a - b);
  return num_list.slice(5); // ← 끝까지 자동
}
```

### 핵심 — `slice`의 두 번째 인자는 옵션

```javascript
const arr = [1, 2, 3, 4, 5, 6, 7];

arr.slice(2); // [3, 4, 5, 6, 7]   ← 2번부터 끝까지
arr.slice(2, 5); // [3, 4, 5]         ← 2번부터 5번 직전까지
arr.slice(2, arr.length); // [3, 4, 5, 6, 7]   ← 위랑 똑같음 (불필요)
```

> **외울 패턴:**
>
> - "n번부터 **끝까지**" → `slice(n)` (length 쓸 필요 X)
> - "n번부터 m번 직전까지" → `slice(n, m)`
> - **`length`를 끝 인자로 쓸 일은 거의 없음.** 생략 = 끝까지가 디폴트.

### 2. 🆕 `slice` vs `splice` — 오늘의 빅보스 😤

> **본인 멘트: "어제 slice, substring, substr 배웠는데 splice가 좀 헷갈리네"**
> 그럴 만함. **이름이 한 글자 차이라 진짜 악마 같음.**

### 한 번에 정리

| 메서드                                 | 원본 변경?     | 리턴값                 | 용도                    |
| -------------------------------------- | -------------- | ---------------------- | ----------------------- |
| `slice(start, end)`                    | ❌ **안 바꿈** | **잘라낸 새 배열**     | 일부 **읽기/복사**      |
| `splice(start, deleteCount, ...items)` | ✅ **바꿈**    | **삭제된 요소들 배열** | 일부 **삭제/삽입/교체** |

### 예시로 비교

```javascript
const a = [1, 2, 3, 4, 5];
const result1 = a.slice(1, 3);
// result1 = [2, 3]
// a       = [1, 2, 3, 4, 5]   ← 그대로

const b = [1, 2, 3, 4, 5];
const result2 = b.splice(1, 3);
// result2 = [2, 3, 4]          ← 삭제된 애들
// b       = [1, 5]             ← 원본이 바뀜!!
```

### `splice`는 삽입/교체도 됨 (인자 추가)

```javascript
const arr = [1, 2, 3, 4, 5];

// 삭제만
arr.splice(1, 2); // [2, 3] 삭제 → arr = [1, 4, 5]

// 삭제 + 삽입
const arr2 = [1, 2, 3, 4, 5];
arr2.splice(1, 2, 99, 100); // [2, 3] 빼고 99, 100 넣음 → arr2 = [1, 99, 100, 4, 5]

// 삽입만 (deleteCount = 0)
const arr3 = [1, 2, 3];
arr3.splice(1, 0, 99); // arr3 = [1, 99, 2, 3]
```

### 🔥 본인이 헤맨 부분

```javascript
cards1 = cards1.splice(0, 1); // ❌ 이거 아닌 거 같더라 → 맞음, 잘못됨
```

> **이유:** `splice`는 **원본을 직접 바꾸고**, 리턴값은 **삭제된 요소 배열**임.
> 그래서 `cards1 = cards1.splice(0, 1)` 하면:
>
> 1. 원본 `cards1`은 첫 요소 빠진 상태로 바뀌고
> 2. 그 결과를 `cards1`에 다시 할당하니까 → `cards1`은 **삭제된 요소 1개짜리 배열**이 됨 ㅋㅋ
> 3. 원래 의도(첫 요소 뺀 배열)랑 정반대 결과

```javascript
const arr = [1, 2, 3, 4, 5];
arr = arr.splice(0, 1);
// arr = [1]   ← 삭제된 거 1개만 남음
// 원래 의도: [2, 3, 4, 5]였을 텐데 정반대
```

### 그래서 올바른 사용

```javascript
// 첫 요소 뺀 배열을 원하면:
cards1.splice(0, 1); // ← 그냥 호출만. 원본이 알아서 바뀜
// 또는
cards1.shift(); // ← 더 깔끔 (아래에서 설명)
// 또는
cards1 = cards1.slice(1); // ← 원본 안 바꾸고 새 배열 받기
```

> **외울 패턴:**
>
> - **slice = sli**ce는 **s**afe (안전, 원본 안 건드림)
> - **splice = sp**lice는 **sp**oil (망침, 원본 바꿈)
> - 결과 받아서 쓰고 싶으면 → `slice`
> - 원본을 직접 수정하고 싶으면 → `splice`
> - **`splice`는 절대 `=`로 다시 할당하지 말 것.** 그냥 호출.

### 3. 🆕 `fill` — 같은 값으로 배열 채우기

[문제: 같은 숫자 반복](https://school.programmers.co.kr/learn/courses/30/lessons/181861?language=javascript)

### 내가 푼 코드 (이중 for)

```javascript
function solution(arr) {
  var answer = [];

  for (var item of arr) {
    for (var i = 0; i < item; i++) {
      answer.push(item);
    }
  }

  return answer;
}
```

### 다른 사람 풀이 — `fill` 사용

```javascript
function solution(arr) {
  return arr.flatMap((n) => new Array(n).fill(n));
}
```

### `fill` 기본 사용법

```javascript
new Array(5).fill(0); // [0, 0, 0, 0, 0]
new Array(3).fill("a"); // ['a', 'a', 'a']
new Array(4).fill(7); // [7, 7, 7, 7]

// 기존 배열에도 사용 가능 (원본 변경 ⚠️)
const arr = [1, 2, 3, 4, 5];
arr.fill(0); // arr = [0, 0, 0, 0, 0]
arr.fill(9, 1, 3); // 인덱스 1 ~ 3 직전을 9로 → [0, 9, 9, 0, 0]
```

### `fill` 자주 쓰는 패턴

```javascript
// 1. 0으로 채워진 길이 n 배열 만들기
new Array(n).fill(0);
Array(n).fill(0); // new 없어도 됨
Array.from({ length: n }, () => 0); // 같은 결과

// 2. 같은 값 n개 만들기
Array(5).fill("hi"); // ['hi', 'hi', 'hi', 'hi', 'hi']

// 3. 2차원 배열 만들 때 주의 ⚠️
Array(3).fill([]); // ❌ 모두 같은 배열 참조!
Array.from({ length: 3 }, () => []); // ✅ 각각 다른 배열
```

### `flatMap` 같이 알기 (위 풀이에 나온 거)

```javascript
[1, 2, 3].flatMap((n) => new Array(n).fill(n));
// 1 → [1]
// 2 → [2, 2]
// 3 → [3, 3, 3]
// flat 합쳐서 → [1, 2, 2, 3, 3, 3]
```

> **외울 패턴:**
>
> - "같은 값 n개" → `Array(n).fill(value)`
> - "0으로 초기화된 배열" → `Array(n).fill(0)`
> - 본인 코드처럼 이중 for 돌리지 말고 `fill`로 한 줄

> **회고:** "요즘 `fill` 자주 나오네"라고 적은 거 → **이게 바로 패턴 인식의 시작.**
> 같은 메서드가 여러 풀이에서 보이면 = 무기로 등록할 타이밍.

## 1단계 — 4문제, 그리고 시간복잡도 첫 경험 🔥

### 4. 최솟값 빼기 — `Math.min(...arr)` 반복 호출의 함정 ⏰

[문제: 제일 작은 수 제거하기](https://school.programmers.co.kr/learn/courses/30/lessons/12935)

### 처음 푼 코드 (느림)

```javascript
function solution(arr) {
  var answer = [];
  if (arr.length === 1) return [-1];

  for (const item of arr) {
    if (item !== Math.min(...arr)) {
      // ← 매 반복마다 Math.min 호출!!
      answer.push(item);
    }
  }

  return answer;
}
```

> **테스트 결과: `통과 (5441.74ms, 41.9MB)`** ← **5.4초??** 미친 ㅋㅋ

### 수정한 코드 (빠름)

```javascript
function solution(arr) {
  if (arr.length === 1) return [-1];

  const min = Math.min(...arr); // ← 한 번만 계산
  return arr.filter((item) => item !== min);
}
```

### 🔥 본인이 직접 발견한 핵심

> 본인 멘트: **"아 `Math.min(...arr)` 이거를 변수에 빼고 했더니 저런 거 안 나오네. 계속 계산해서 시간이 오래 걸렸나 보다"**
>
> **정확함.** 이게 바로 **시간복잡도 직관**의 첫 단추.

### 왜 그렇게 느렸나? — O(n²) 문제

```javascript
// arr 길이가 n일 때
for(const item of arr) {              // n번 반복
    if(item !== Math.min(...arr)) {   // 매번 n개 다 훑음
        ...
    }
}
// → n × n = n² 번 연산

// 수정 후
const min = Math.min(...arr)          // n번 (1회)
arr.filter(...)                       // n번
// → 2n 번 연산
```

> 배열 길이가 10,000이면:
>
> - 수정 전: 10,000 × 10,000 = **1억 번**
> - 수정 후: 10,000 × 2 = **2만 번**
> - **5,000배 차이** — 그래서 5초 걸림.

### 외울 룰 — 반복문 안에서 "변하지 않는 값"은 빼라

```javascript
// ❌ 나쁜 패턴 — 매번 다시 계산
for (let i = 0; i < arr.length; i++) {
    if (arr[i] === Math.max(...arr)) { ... }
}

// ✅ 좋은 패턴 — 한 번만 계산
const max = Math.max(...arr)
for (let i = 0; i < arr.length; i++) {
    if (arr[i] === max) { ... }
}
```

> **외울 패턴:**
>
> - **반복문 안에서 호출되는 함수는 일단 의심하라.** 한 번만 계산해도 되는 거면 밖으로.
> - `Math.min(...arr)`, `Math.max(...arr)`, `arr.length`(가끔), `arr.includes(...)` 등
> - **이게 시간복잡도 최적화의 가장 기본.**

> 🎯 **오늘의 진짜 수확.** 책으로 배운 게 아니라 **본인이 직접 5초짜리 결과 보고 깨달음.**
> 이 경험은 절대 안 까먹음. 두고두고 본인 무기 됨.

### 🆕 `shift()` — 첫 요소 빼기

> 본인 멘트: "다른 사람 풀이 보니까 `shift()`도 있네 이건 뭐지"

```javascript
const arr = [1, 2, 3, 4, 5];
const first = arr.shift();
// first = 1
// arr   = [2, 3, 4, 5]   ← 원본이 바뀜
```

### `shift` / `unshift` / `pop` / `push` — 한 번에 정리

| 메서드       | 위치 | 동작 | 리턴값      |
| ------------ | ---- | ---- | ----------- |
| `push(x)`    | 뒤   | 추가 | 새 길이     |
| `pop()`      | 뒤   | 제거 | 제거된 요소 |
| `unshift(x)` | 앞   | 추가 | 새 길이     |
| `shift()`    | 앞   | 제거 | 제거된 요소 |

```javascript
const arr = [2, 3, 4];

arr.push(5); // arr = [2, 3, 4, 5],  return 4
arr.pop(); // arr = [2, 3, 4],     return 5
arr.unshift(1); // arr = [1, 2, 3, 4],  return 4
arr.shift(); // arr = [2, 3, 4],     return 1
```

### `splice(0, 1)` vs `shift()` — 같은 결과, 다른 표현

```javascript
arr.splice(0, 1); // 첫 요소 제거 (리턴: [제거된 요소])
arr
  .shift() // 첫 요소 제거 (리턴: 제거된 요소 자체)

  [
    // 차이: splice는 배열 리턴, shift는 요소 자체 리턴
    (1, 2, 3)
  ].splice(0, 1) // [1]   ← 배열
  [(1, 2, 3)].shift(); // 1     ← 값
```

> **외울 패턴:**
>
> - 첫 요소 그냥 빼고 싶으면 → **`shift()` 가 더 직관적**
> - 여러 개 빼거나 중간에서 빼야 하면 → `splice`
> - 의도가 "큐(앞에서 꺼내기)"면 → `shift`
> - 의도가 "특정 위치 조작"이면 → `splice`

### 5. 카드 뭉치 — `splice` 처음 실전 적용

[문제: 카드 뭉치](https://school.programmers.co.kr/learn/courses/30/lessons/159994)

### 푼 코드

```javascript
function solution(cards1, cards2, goal) {
  var answer = "Yes";

  for (const item of goal) {
    if (cards1[0] === item) {
      cards1.splice(0, 1);
    } else if (cards2[0] === item) {
      cards2.splice(0, 1);
    } else {
      answer = "No";
    }
  }

  return answer;
}
```

### 위에서 정리한 거 적용 — `shift()`로 바꾸면 더 깔끔

```javascript
function solution(cards1, cards2, goal) {
  for (const item of goal) {
    if (cards1[0] === item) {
      cards1.shift();
    } else if (cards2[0] === item) {
      cards2.shift();
    } else {
      return "No"; // ← 여기서 바로 return
    }
  }
  return "Yes";
}
```

### 개선 포인트 2개

1. **`splice(0,1)` → `shift()`** — 의도가 "큐에서 꺼내기"라 `shift`가 적절
2. **`No` 만나면 즉시 return** — `answer` 변수 안 만들어도 됨 (early return)

### 미니 회고 — `splice` 쓸 때 본인이 헷갈렸던 부분 다시 정리

```javascript
// ❌ 본인이 처음 시도한 거
cards1 = cards1.splice(0, 1);
// → cards1이 [삭제된 요소]가 됨. 의도랑 정반대

// ✅ 올바른 사용
cards1.splice(0, 1);
// → 호출만 하면 cards1 자체가 바뀜
```

> **`splice`의 핵심: "리턴값 받지 말고, 호출만 해라."** (대부분의 경우)
> 리턴값(=삭제된 요소들)이 필요한 특수한 경우에만 받기.

## 🤔 오늘의 회고 — 시간복잡도 첫 자각의 날

### 오늘의 빅이벤트 = `Math.min(...arr)` 5초 사건

> 양치기로 손 익히는 거랑 별개로, **오늘은 진짜 "사고의 점프"가 있었음.**
>
> "왜 5초나 걸리지?" → 코드 다시 봄 → "어 이거 매번 계산하잖아?" → 변수로 빼봄 → 0.x초
>
> 이게 바로 **알고리즘 사고의 시작.** 책 읽어서 배운 게 아니라 **직접 측정 결과 보고 깨달은 거**라 박힘 강도 만렙.

### 어제(9일차) → 오늘(10일차) 패턴 비교

| 항목      | 9일차                               | 10일차                                         |
| --------- | ----------------------------------- | ---------------------------------------------- |
| 컨디션    | 망함 (목/감기/스트레스)             | 회복                                           |
| 전략      | 양치기 (1단계 1페이지)              | 양치기 + 1단계 도전                            |
| 새 무기   | `Math.abs`, `slice(-n)`             | `splice`, `shift`, `fill`, **시간복잡도 직관** |
| 회고 효과 | 어제 거 훑기 → `toUpperCase` 재사용 | 어제 거 훑기 → `slice` 끝 인자 생략 발견       |

> **회고 시스템이 본격적으로 굴러가기 시작함.** 어제 정리한 게 오늘 자연스럽게 연결됨.

### 패턴 — "다른 사람 풀이에서 자주 나오는 메서드"가 무기

오늘 본 것:

- 어제: `Math.abs`, `slice(-n)`, `localeCompare`
- 오늘: `fill`, `splice`, `shift`

> **양치기의 진짜 가치 = 다른 사람 풀이 보면서 자주 보이는 패턴 줍기.**
> 한 번만 보면 "흠~" 하고 지나가는데, 두세 번 보면 "이거 무기네" 하고 박힘.

### 🆕 새로 배운 거

- **`splice(start, deleteCount, ...items)` — 원본 변경, 삭제된 거 리턴**
- **`shift()` / `unshift()` / `pop()` / `push()` — 앞뒤 추가/제거 4총사**
- **`Array(n).fill(value)` — 같은 값으로 배열 채우기**
- **`flatMap` — map + flat 합친 거** (위 fill 풀이에서)
- 🔥 **반복문 안에서 변하지 않는 값은 변수로 빼라** = 시간복잡도 절약

### 🔁 다시 박은 패턴

- `Math.min(...arr)` / `Math.max(...arr)` — 어제 `Math.min/max` 정리한 게 오늘 등판
- 원본 변경 메서드는 `=`로 재할당하지 말 것 (`splice`, `sort` 등)

### ⚠️ 오늘의 실수 / 헤맨 거

- `cards1 = cards1.splice(0, 1)` — `splice` 리턴값 잘못 이해함 (삭제된 요소 배열을 변수에 다시 할당)
- `Math.min(...arr)` 반복 호출로 5초 — 시간복잡도 무지의 결과
- → **둘 다 본인이 직접 발견하고 수정함.** 이게 진짜 학습.

### 🎯 어제 다짐 실천 점검

- ✅ "최근 회고 5분 훑기" → 함 (어제 `slice` 정리한 게 오늘 `slice(n)` 발견에 연결)
- ✅ "양치기 + 새 무기 줍기" → 정확히 그렇게 함
- ✅ "지문 천천히 읽기" → 컨디션 좋아져서 자연스럽게 됨

### 🎯 내일을 위한 다짐

- **반복문 안에서 같은 함수 호출하고 있지 않은지 항상 체크.** 이거 룰로 박자.
- **원본 변경 메서드 (`splice`, `sort`, `reverse`, `push`, `shift` 등)는 `=`로 재할당 금지.** 그냥 호출.
- 회고 훑기 계속 유지 — **시스템이 돌아가고 있음.**
