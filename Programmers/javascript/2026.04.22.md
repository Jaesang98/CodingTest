### 복습 — split / join

**split**

- `split("")` — 문자열을 한 글자씩 잘라서 배열로 변환
- `split("기준")` — 기준 문자열을 중심으로 나눠서 배열로 변환

**join**

- `join("")` — 배열 원소를 전부 붙여서 하나의 문자열로
- `join("구분자")` — 각 원소 사이에 구분자를 넣어서 연결

### 문제: 배열 자르기 ([링크](https://school.programmers.co.kr/learn/courses/30/lessons/120833))

**새로 배운 개념 — `slice(index1, index2)`**

- `index1` 위치는 포함, `index2` 위치는 미포함
- 그래서 num2번째 인덱스까지 **포함**하려면 `index2`에 1을 더해줘야 함

```javascript
[1, 2, 3, 4, 5]
  .slice(1, 3) // [2, 3] → index 1, 2만 나옴 (3 제외)
  [(1, 2, 3, 4, 5)].slice(1, 4); // [2, 3, 4] → 1을 더해야 4까지 포함
```

## 1단계 문제 풀이

### 문제: K번째 수 ([링크](https://school.programmers.co.kr/learn/courses/30/lessons/42748?language=javascript))

**소요 시간:** 혼자 시도 → 반례 확인 후 수정

### 풀이 흐름

| 버전    | 핵심 변화                                             |
| ------- | ----------------------------------------------------- |
| 1차     | 혼자 풀이 — `.sort()` 기본 사용, 두 자리 수 정렬 오류 |
| **2차** | 반례 확인 후 `sort((a, b) => a - b)` 추가 → 통과 ✅   |

---

### 1차 시도 — sort 기본 사용 오류

> `.sort()`를 인자 없이 쓰면 숫자를 문자열로 비교해서 10이 6보다 앞에 오는 버그 발생.
> ex) `[10, 9, 8, 7, 6].sort()` → `[10, 6, 7, 8, 9]`

```javascript
function solution(array, commands) {
  var result = [];

  for (var i = 0; i < commands.length; i++) {
    var filter = array.slice(commands[i][0] - 1, commands[i][1]).sort();
    result[i] = filter[commands[i][2] - 1];
  }

  return result;
}
```

> 테스트 2 실패. 처음엔 인덱스 문제인 줄 알고 `commands[i][0]-1 ?? commands[i][0]` 같은 시도를 해봤지만 원인은 인덱스가 아니라 **sort의 기본 동작**이었음.

---

### 반례 확인

> 입력값: `[10, 2], [[1, 2, 1]]` → 기댓값 `[2]`
> `.sort()` 결과: `[10, 2]` → 문자열 기준 정렬이라 10이 2 앞에 그대로 남음

---

### 2차 시도 (최종) — `sort((a, b) => a - b)` 추가 ✅ 통과

```javascript
function solution(array, commands) {
  var result = [];

  for (var i = 0; i < commands.length; i++) {
    var filter = array.slice(commands[i][0] - 1, commands[i][1]);
    filter = filter.sort((a, b) => {
      return a - b;
    });
    result[i] = filter[commands[i][2] - 1];
  }

  return result;
}
```

---

### 도중에 생긴 의문

```javascript
for (var i in commands) {
  return i; // "0", "1", "2" ... 만 나옴
}
```

> `for...in`을 배열에 쓰면 값이 아니라 **인덱스만 반환됨 — 그것도 숫자가 아니라 문자열 "0", "1", "2"로**.
> 객체(딕셔너리)에 쓰면 키가 나오는 것처럼, 배열에서는 인덱스가 키 역할을 하는 구조.

## 오늘의 핵심 개념 정리

### `slice` 범위 주의

```javascript
// num1=1, num2=3 일 때 index 1~3 포함해서 뽑으려면
array.slice(num1, num2 + 1); // ✅ index2는 미포함이라 +1 필요
```

### 숫자 정렬 — `sort` 주의사항

```javascript
// ❌ 기본 sort — 문자열(유니코드) 기준
[10, 9, 2]
  .sort() // [10, 2, 9]

  [
    // ✅ 숫자 정렬 — 비교 함수 필수
    (10, 9, 2)
  ].sort((a, b) => a - b); // [2, 9, 10]
```

**왜 `.sort()`는 안 되는가?**

> `.sort()`는 기본적으로 요소를 **문자열로 변환한 뒤 유니코드 순서로** 비교함.
> "10"은 문자열 기준으로 "2"보다 앞에 오기 때문에 숫자로는 틀린 결과가 나옴.

**왜 `.sort((a, b) => a - b)`는 되는가?**

> 비교 함수의 반환값으로 정렬 순서를 직접 지정하는 방식.
>
> - `a - b < 0` → a를 b 앞에
> - `a - b > 0` → b를 a 앞에
> - `a - b === 0` → 순서 유지
>
> 숫자끼리 빼기 때문에 크기 비교가 정확하게 동작함.

| 방식                     | 정렬 기준        | 주의                       |
| ------------------------ | ---------------- | -------------------------- |
| `.sort()`                | 문자열(유니코드) | 두 자리 이상 숫자에서 오류 |
| `.sort((a, b) => a - b)` | 숫자 오름차순    | ✅ 안전                    |

### `for...in` vs `for...of`

| 구문       | 배열에서 반환                            | 객체에서 반환 |
| ---------- | ---------------------------------------- | ------------- |
| `for...in` | 인덱스 — **문자열 타입** ("0", "1", "2") | 키            |
| `for...of` | 값                                       | ❌ 사용 불가  |

```javascript
const arr = ["사과", "바나나", "딸기"];
const obj = { name: "김철수", age: 25 };

for (var i in arr) console.log(i); // "0" "1" "2" (문자열!)
for (var v of arr) console.log(v); // "사과" "바나나" "딸기"
for (var k in obj) console.log(k); // "name" "age"
for (var v of obj) console.log(v); // ❌ TypeError: obj is not iterable
```

> 배열 순회할 때 값이 필요하면 `for...of`, 인덱스가 필요하면 `for (var i = 0; ...)` 쓰는 게 안전.
> `for...in`은 **객체 키 꺼낼 때** 쓰는 용도로 기억하기.
